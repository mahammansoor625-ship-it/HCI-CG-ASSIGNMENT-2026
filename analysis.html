Parallel Analysis: Setting an Alarm on a Smartphone

Task: Set an alarm for 6:00 AM

Norman’s Stage Real-World Example GOMS Interpretation Alignment Analysis
Goal ↔ GOAL Need to wake up at 6:00 AM Set alarm for 6:00 AM Both models begin with intention
Plan ↔ METHOD Open Clock app and add alarm Method: Open → Alarm → Add Conceptual alignment
Specify Select hour, minute, and AM Not explicitly modeled Divergence in cognitive detail
Execute ↔ OPERATOR Scroll and tap “Save” Operators: Scroll, Tap Direct execution match
Evaluate Alarm shows active status Outside GOMS scope Norman includes feedback loop

Conclusion

GOMS efficiently models goals and execution for routine tasks but omits specification and feedback. Norman’s Action Cycle provides a more complete framework by explicitly addressing perception, interpretation, and evaluation, making it more suitable for real-world usability analysis.
