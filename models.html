Norman’s Action Cycle in HCI (Chapter 3)

Definition:
In Human–Computer Interaction, norms refer to accepted rules and expectations that guide how users interact with systems. Norman’s Action Cycle explains how users translate goals into actions and evaluate outcomes.

Stages of Norman’s Action Cycle:

  1. Goal Formation: Decide what to achieve
  2. Intention Formation: Plan how to achieve it
  3. Action Execution: Perform the action
  4. System Response: System reacts
  5. Perception: Observe the response
  6. Interpretation: Understand the response
  7. Evaluation: Confirm goal achievement

Design Note:
Interfaces that align with established norms reduce user confusion and cognitive effort.

GOMS Model in HCI (Chapter 12)

Overview:
GOMS is a predictive model used to analyze how users perform tasks. It estimates execution time and interaction efficiency.

GOMS stands for:

  • Goals
  • Operators
  • Methods
  • Selection Rules

Key Components:

  • Goals: Desired outcomes
  • Operators: Basic actions (clicks, keystrokes)
  • Methods: Steps used to reach a goal
  • Selection Rules: Decide which method to use

Importance:

  • Predicts task performance
  • Compares interface efficiency
  • Useful for expert-user analysis

Limitations:

  • Assumes expert users
  • Does not model errors or learning
  • Unsuitable for exploratory tasks
