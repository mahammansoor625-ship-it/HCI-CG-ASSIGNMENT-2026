Part A: Find Mistakes (Identify + Fix)

UI Element / Issue Wrong HTML Correct UI Element Correct HTML
Navigation links not wrapped in <li> <nav> <ul> <a href="home.html">Home</a> <a href="about.html">About</a> </ul> </nav> Navigation list with list items <nav aria-label="Primary navigation"> <ul> <li><a href="home.html">Home</a></li> <li><a href="about.html">About</a></li> </ul> </nav>
Label not associated with input <label>Email</label> <input type="email"> Label linked to input <label for="email">Email</label> <input type="email" id="email">
Incorrect heading order <h1>User Research</h1> <h3>Interview Findings</h3> Proper heading hierarchy <h1>User Research</h1> <h2>Interview Findings</h2>
Table missing caption and headers <table> <tr> <td>Model</td> <td>Purpose</td> </tr> </table> Accessible table structure <table> <caption>HCI Models Overview</caption> <thead> <tr> <th scope="col">Model</th> <th scope="col">Purpose</th> </tr> </thead> </table>

Part B: True / False — Answers

  • Every link inside <ul> must be inside <li>. — ✅ True
  • A page may have multiple <h1> to make text larger. — ❌ False
  • aria-current="page" marks the active link. — ✅ True
  • A form is accessible without labels. — ❌ False
  • KLM predicts expert, error-free performance. — ✅ True

Part C: Short Answer — Answers

Semantic vs Generic elements

Semantic elements describe meaning and structure (e.g., <nav>, <article>), while generic elements (<div>, <span>) are used only for grouping or styling.

Why KLM ignores high-level planning?

KLM models low-level physical actions and assumes skilled users. High-level thinking varies between users and is excluded for consistent prediction.
